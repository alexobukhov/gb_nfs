Как организовать клиент-серверное взаимодействие?
Клиент-серверное взаимодействие целесообразно организовать на уровне TCP. Для этого можно использовать SFTP подключение.

Как и в каком виде передавать файлы?
Файлы передаются можно передавать с помощью кодировки Base64, которая используется для кодирования массива байт []byte.

Как пересылать большие файлы?
Большие файлы целесообразно делить на несколько частей и собирать уже в месте доставки/получения.

Как пересылать служебные команды?
Служебные команды целесообразно посылать с помощью служебных сообщений через Rest API.

Что хранить в базе данных?
В базе данных целесообразно хранить относительную ссылку на файл.

Как передавать структуру каталогов/файлов?
Структуру каталогов/фалов целесообразно передавать в виде относительных ссылок.

Какую библиотеку использовать для сетевого взаимодействия: java.io, java.nio, Netty?
Java NIO и Netty

Для взаимодействия планируется реализовать приложение с использованием Spring Boot с использованием следующих библиотек:
Spring Data
Spring Security
Spring Web

Серверное приложение по сути будет представляеть собой сервис, который с помощью Rest API будет предоставлять методы для
получения информации о файлах и структуре каталогов и для изменения собственно самих файлов и каталогов.

Клиентское приложение планируется реализовать на платформе Vaadin (Spring под капотом), в котором планируется использование
следующих библиотек:
Spring Security
